# 3. Advanced Analysis and Visualization

## Advanced Analysis Workflow

Once you have a `ComplexMap` object, the package provides several
functions for deeper analysis and interpretation of the network
structure. This vignette covers:

1.  **Optimizing Parameters**: Using
    [`evaluateComplexes()`](https://zqzneptune.github.io/ComplexMap/reference/evaluateComplexes.md)
    to tune the `refineComplexList` step.
2.  **Summarizing & Querying Themes**: A streamlined workflow using
    [`summarizeThemes()`](https://zqzneptune.github.io/ComplexMap/reference/summarizeThemes.md)
    and
    [`queryMap()`](https://zqzneptune.github.io/ComplexMap/reference/queryMap.md)
    to identify and investigate biological clusters.

``` r

library(ComplexMap)
library(dplyr)
```

### 1. Optimizing Refinement with `evaluateComplexes`

The `mergeThreshold` in `refineComplexList` determines how aggressively
redundant complexes are merged. The default is a conservative
**Jaccard** similarity of 0.90 to preserve biological variants.

The
[`evaluateComplexes()`](https://zqzneptune.github.io/ComplexMap/reference/evaluateComplexes.md)
function can score a refined complex list against a gold standard,
helping you tune this parameter.

``` r

# Load the package's demo and reference data
data(demoComplexes)
data(referenceComplexes)

# 1. Run Refinement
refinement_output <- refineComplexList(
  demoComplexes,
  mergeThreshold = 0.90,
  similarityMethod = "jaccard",
  verbose = FALSE
)

# 2. Evaluate
# Pass the $refinedComplexes element to the evaluation function
metrics <- evaluateComplexes(
  predictedComplexes = refinement_output$refinedComplexes,
  referenceComplexes = referenceComplexes,
  nCores = 2, # Use multiple cores if available
  verbose = FALSE
)

# Display the resulting metrics
knitr::kable(as.data.frame(metrics))
```

|       PPV |        Sn |       Acc |       MMR |
|----------:|----------:|----------:|----------:|
| 0.8005051 | 0.2510583 | 0.4483006 | 0.0940346 |

For more systematic tuning, see the documentation for
[`benchmarkParameters()`](https://zqzneptune.github.io/ComplexMap/reference/benchmarkParameters.md),
which automates this process over a range of thresholds.

### 2. A Streamlined Workflow for Theme Analysis

A key goal of `ComplexMap` is to organize complexes into a functional
landscape.
[`summarizeThemes()`](https://zqzneptune.github.io/ComplexMap/reference/summarizeThemes.md)
uses community detection to find densely connected physical
neighborhoods (themes) and, by default, adds this information directly
to your `ComplexMap` object. This makes querying by theme simple and
intuitive.

First, let’s generate a map object for this example.

``` r

gmtPath <- getExampleGmt()
gmt <- getGmtFromFile(gmtPath, verbose = FALSE)

cm_obj_adv <- createComplexMap(
  complexList = demoComplexes,
  gmt = gmt,
  mergeThreshold = 0.85, 
  verbose = FALSE
)
```

#### Step 2.1: Summarize Themes and Add to Object

Now, we run
[`summarizeThemes()`](https://zqzneptune.github.io/ComplexMap/reference/summarizeThemes.md).
The default `add_to_object = TRUE` returns the modified `ComplexMap`
object, now containing theme information in its node table.

``` r

# Ensure reproducibility of community detection
set.seed(123)

# This single command finds themes AND adds them to the object
cm_obj_adv <- summarizeThemes(cm_obj_adv, verbose = FALSE)

# We can inspect the new columns in the node table
getNodeTable(cm_obj_adv) %>%
  select(complexId, primaryFunctionalDomain, themeId, themeLabel) %>%
  head() %>%
  knitr::kable()
```
